All About Github

    Q. What is a version control system?

    A. A class of systems responsible for managing changes to computer programs, documents, large web sites, or other collections of information.

    Q. What is Git?

    A. A distributed version control system, which means that a local clone of the project is a complete version control repository.

    Q. What is Github?

    A. A platform and cloud-based service for software development and version control using Git, allowing developers to store and manage their code.

    Q. What is the difference between Git and Github?

    A. Git is a revision control system, a tool to manage your source code history. GitHub is a hosting service for Git repositories.

    Q. Who started Github and how was it started?

    A. The GitHub service was developed by Chris Wanstrath, P. J. Hyett, Tom Preston-Werner, and Scott Chacon using Ruby on Rails, and started in February 2008. GitHub, Inc. was originally a flat organization with no middle managers; in other words, "everyone is a manager" (self-management).[17] Employees could choose to work on projects that interested them (open allocation), but the chief executive set salaries.

    Q. What company owns it now?

    A. Microsoft now owns GitHub. Microsoft acquired the developer platform for $7.5 billion in a deal.

    Q. How much does a Github account cost?

    A. GitHub Copilot for Individuals cost $10 USD/month or $100 USD/year per seat. GitHub Copilot for Business cost $19 per seat, per month.

    Q. What is an Octocat?

    A. The mascot of the source-code hosting service that is a hybrid of a cat and an octopus.

Git/Github terms

  1. Repository: A Git repository is a virtual storage of your project.
  2. Commit: A individual change to a file (or set of files).
  3. Fork: A copy of an existing repository in which the new owner disconnects the codebase from previous committers.
  4. Push: Transfers or pushes the commit, which is made on a local branch in your computer to a remote repository like GitHub.
  5. Pull Requests: Specific requests to merge changes from one branch into another, often for code review and integration before merging.
  6. Workflows: A recipe or recommendation for how to use Git to accomplish work in a consistent and productive manner.
  7. Issues: Tracks ideas, feedback, tasks, or bugs for work on GitHub.
  8. Raw Button: Allows you to download the raw contents of a file. The raw URL is for that particular version of the file.
  9. Blame Button: A command that displays authorship information regarding the latest changes to every line in a file.
